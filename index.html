Open Data Portal and Web Apps

This page shows examples of how Detroit's Open Data Portal as well as web-based applications could be better integrated in the menus of Detroitmi.gov in order to be more accessible to users. There is also a plan to create a new internal page in Detroitmi.gov that would showcase and link to all available web apps.

Additions to the Hamburger Menu

In the main menu of Detroitmi.gov, we could add 1 or 2 additional links. One external link that could lead to the ESRI Open Data Portal, and one that could link to a newly created internal Detroitmi.gov page which houses all web apps. One or both are possible.

Many other cities do not link to their open data portal on the front page of their website, however, web applications that help people complete common tasks are usually linked to on the front page.

Right now the menu structure is based on what is most accessed or searched for, however this approach means that tools that could save residents a lot of time, might not be discovered if they don’t know to look for them. If we think that these resources are potentially valuable to residents, and could help unburden the FOIA team this will be one of the more effective spots to elevate this information.

Effort required: This will require minimal engineering effort to implement. We will just need a formal request from Alexis/Graham to the web team for this content change. This section is traditionally powered by analytics, and they would love buyin for pivoting from that approach somewhat.

Additions to the "Working for You" Menu

In addition to the 2 links added to the hamburger menu, we could also add a Technology section to the Working for You interactive menu on the front page. If the Hamburger menu is not the preferred spot, this could also be an alternate option. From this section, we would place the same links to the external Open Data Portal and the internal Web Apps page.

Effort required: This will require minimal engineering effort to implement. We will not need a formal request from Alexis/Graham to the web team for this content change as I understand it. We would need a formal request from Beth to the web team.

New Web Apps Page

If we do decide to create and link to a new Web Apps page on the front page of Detroitmi.gov, this would allow for us to create a portal for all web applications built by city teams. Right now some of these applications only live on the open data portal. Additionally our Web Apps tag on detroitmi.gov does not pull together all web applications used across the city. It takes several clicks to find information on the Parking App for example. Let’s make a one stop searchable shop for all our residents looking for digital resources!

Effort required: This will require minimal engineering effort to implement, but will take some time to gather all the content and find thumbnail photos for each web app. To help with that, Juan Carlos from the IET team would be able to implement without requiring additional time from the web team if helpful. We will just need a formal request from Alexis/Graham to the web team for this content change. This section is traditionally powered by analytics, and they would love buy-in for pivoting from that approach somewhat.

New Department Menu Item

We could also begin to further incorporate Web Apps into the menus on Department pages. Because of issues with the current Drupal templates for web apps and hosting (IET apps currently hosted on Github), not all Web Apps in the city appear automatically when tagged.

Effort Required: While the design change might feel fairly simple this would require significantly more dev time to implement. The overall drupal templates would have to shift and the template structure for web apps would need to be investigated and updated as well. Additionally, we may need to look into some infrastructure improvements so that apps that are not currently hosted on the same server as Detroitmi.gov could still potentially be referenced. While this seems like a longer term goal, it will likely take significant time and effort.
